# 24 - Convenções de Código

## Geral

* Usar TypeScript.
* Manter módulos coesos.
* Evitar lógica de negócio duplicada.
* Validar entradas no backend.
* Tratar erros de forma explícita.

## Nomes

* Entidades em inglês no código.
* Textos de interface em português.
* Arquivos e pastas com nomes consistentes com o framework utilizado.

## API

* Rotas REST no plural.
* DTOs para entrada e saída.
* Guards para autenticação e autorização.
* Eventos WebSocket com nomes claros.

## Frontend

* Componentes reutilizáveis.
* Formulários com validação.
* Estados de loading, erro e vazio.
* Separar chamadas de API da apresentação.

## Testes

* Testar regras de negócio.
* Testar permissões.
* Testar fluxos críticos da operação.
