# 17 - Design System

## Objetivo

Definir padrões visuais e componentes reutilizáveis para manter consistência entre o painel administrativo, portal do cliente e app do motorista.

## Componentes base

* Botões.
* Campos de formulário.
* Tabelas.
* Filtros.
* Badges de estado.
* Modais.
* Toasts.
* Cards de indicadores.
* Mapas.

## Estados visuais

* Sucesso.
* Atenção.
* Erro.
* Informação.
* Inativo.

## Badges de carga

* Criada
* Planeada
* Aguardando Recolha
* Recolhida
* Em Trânsito
* Próxima do Destino
* Entregue
* Cancelada
* Com Incidente

## Princípios de interface

* Interface simples e responsiva.
* Informação operacional fácil de ler.
* Ações críticas com confirmação.
* Estados de carregamento e erro visíveis.
